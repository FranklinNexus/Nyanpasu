## 📚 核心文档

### 1. `README.md` - 项目主页 ⭐
**用途**: GitHub 仓库的门面，介绍项目
**内容**:
- 项目简介和特性
- 截图展示区
- 下载链接
- 构建说明
- 路线图

- 开发者信息

**何时使用**: 发布到 GitHub 时，这是第一个被看到的文件

---

### 2. `LICENSE` - 开源许可证 📄
**用途**: 声明项目使用 MIT 许可证
**内容**:
- MIT 许可证全文
- 版权声明（2026 KuroshiMira）

**何时使用**: 
- 发布开源项目必需
- 允许他人自由使用、修改、分发你的代码

---

### 3. `.gitignore` - Git 忽略文件 🚫
**用途**: 告诉 Git 哪些文件不要上传
**内容**:
- 忽略编译文件（APK、build 文件夹）
- 忽略 IDE 配置（.idea）
- 忽略密钥库文件（*.jks）

**何时使用**: 第一次 `git init` 时就需要

## 📱 应用商店资料

### 6. `STORE_LISTING.md` - 商店展示资料 🛍️
**用途**: Google Play 上架所需的所有文案
**内容**:
- 应用名称
- 简短描述（80 字符）
- 完整描述（4000 字符）
- 分类和标签
- 图形资源要求
- 内容分级指南
- 发布说明模板
- 推广文案（Twitter、Reddit）

**何时使用**: 
- 准备 Google Play 上架资料时
- 需要复制粘贴各种描述时

---

### 7. `PRIVACY_POLICY.md` - 隐私政策 🔒
**用途**: Google Play 必需的隐私政策
**内容**:
- 数据收集声明（不收集任何数据）
- 权限说明
- 第三方服务说明
- 联系方式

**何时使用**: 
- 上传到网上（GitHub Pages 或你的博客）
- 在 Google Play 中填写隐私政策链接

**重要**: Google Play 要求必须有隐私政策 URL！

- 截图工具推荐
- 后期处理建议
- 视频录制指南（可选）

**何时使用**: 
- 准备上传 Google Play 前
- 需要拍摄应用截图时

## 📋 使用流程建议

### 首次发布流程

```
第一阶段：准备工作（今天）
1. ✅ 阅读 QUICK_START.md（了解整体流程）
2. ✅ 按照 SCREENSHOTS_GUIDE.md 拍摄截图
3. ✅ 检查 STORE_LISTING.md，准备文案

第二阶段：GitHub 发布（今天）
4. ✅ 按照 QUICK_START.md 步骤 2，发布到 GitHub
5. ✅ 上传 PRIVACY_POLICY.md 到 GitHub Pages

第三阶段：生成 APK（今天或明天）
6. ✅ 按照 QUICK_START.md 步骤 3，生成签名 APK
7. ✅ 备份密钥库文件！！！

第四阶段：Google Play 上传（等开发者账号审核通过后）
8. ✅ 创建 Google Play 开发者账号（支付 $25）
9. ✅ 按照 PUBLISH_GUIDE.md 步骤 4，上传应用
10. ✅ 复制 STORE_LISTING.md 中的文案

第五阶段：等待审核（1-7 天）
11. ✅ 监控邮箱，等待审核结果
12. ✅ 如被拒绝，根据原因修改后重新提交

第六阶段：发布成功 🎉
13. ✅ 更新 README.md，添加 Google Play 链接
14. ✅ 创建 GitHub Release（tag: v1.0.0）
15. ✅ 在社交媒体宣传（使用 STORE_LISTING.md 中的推广文案）
```
